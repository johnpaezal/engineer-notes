# 00-software-engineering / 01-process / sdlc.md
# ============================================================
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What is the SDLC?	Software Development Lifecycle: the phases to take software from idea to production.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What are the 7 phases of the SDLC?	Planning → Analysis → Design → Development → Testing → Deployment → Maintenance.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What happens in the Planning phase?	Define scope, goals, feasibility, and resources.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What happens in the Analysis phase?	Gather requirements and understand what to build.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What happens in the Design phase?	Define architecture, database schema, and UI wireframes.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What happens in the Maintenance phase?	Bug fixes, updates, and new features after release.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What is the Waterfall model?	Sequential SDLC: each phase ends before the next starts. Predictable but inflexible.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	What is the Agile model?	Iterative SDLC: short cycles (sprints) deliver working software early and often.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	Waterfall vs Agile?	<table border="1" cellpadding="6"><tr><th>Waterfall</th><th>Agile</th></tr><tr><td>Sequential, fixed scope</td><td>Iterative, embraces change</td></tr><tr><td>Delivers at the end</td><td>Delivers every sprint</td></tr></table>
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	When to use Waterfall?	Fixed-scope projects or regulated industries where change is expensive.
AwsEngineer::SoftwareEngineering::Process::SDLC	MiPlantilla	When to use Agile?	Products with unclear or changing requirements.

AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a User Need?	What the user actually wants — the underlying problem.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is an Articulated need?	What people say and think they want.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is an Observable need?	What people actually do and use.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Tacit need?	Conscious but hard to express in words.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Latent need?	Not conscious or not recognized by the user.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is the Interview elicitation technique?	One-on-one with stakeholders: deep insight but slow.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is the Survey elicitation technique?	Wide reach with shallow data; good for many users.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is the Observation elicitation technique?	Watch users in real context; captures tacit needs.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Functional Requirement?	A specific behavior or function the system must do.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Non-Functional Requirement?	A quality attribute (performance, security, scalability, etc.).
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	Functional vs Non-Functional requirement?	<table border="1" cellpadding="6"><tr><th>Functional</th><th>Non-Functional</th></tr><tr><td>What it does</td><td>How well it does it</td></tr></table>
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a User Story?	A requirement written from the user's perspective: "As a... I want... so that..."
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Use Case?	An actor-driven flow describing how a user interacts with the system.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	User Story vs Use Case?	<table border="1" cellpadding="6"><tr><th>User Story</th><th>Use Case</th></tr><tr><td>Short, user perspective</td><td>Detailed actor-driven flow</td></tr></table>
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is Acceptance Criteria?	Conditions a user story must meet to be considered done.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is a Story Point?	Relative unit of effort or complexity, not hours.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is Velocity?	Average story points completed (DONE) per sprint.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	Why use Fibonacci for story points?	Non-linear scale reflects growing uncertainty for larger tasks.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is Planning Poker?	Team estimation: each member secretly picks a card, reveal, discuss, re-estimate.
AwsEngineer::SoftwareEngineering::Process::Requirements	MiPlantilla	What is an Epic?	A large user story broken down into smaller stories.

AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What is the DELETE method for?	Remove a resource.
AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	PUT vs PATCH?	<table border="1" cellpadding="6"><tr><th>PUT</th><th>PATCH</th></tr><tr><td>Replace whole resource</td><td>Update only some fields</td></tr></table>
AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What does 200 OK mean?	Request succeeded.
AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What does 201 Created mean?	Resource was created successfully.
AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What does 204 No Content mean?	Success but no response body.
AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What does 400 Bad Request mean?	Invalid or malformed input from the client.

AwsEngineer::SoftwareEngineering::Design::RestAPI	MiPlantilla	What does 403 Forbidden mean?	Authenticated but not allowed to access this resource.

# 00-software-engineering / 02-design / http-api-deep.md
# ============================================================
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is a Safe HTTP method?	One that doesn't change server state (GET, HEAD).
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is an Idempotent HTTP method?	Same effect if called N times (GET, PUT, DELETE, HEAD).
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	Why is POST not idempotent?	Two POSTs create two distinct resources.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	Why is DELETE idempotent?	A second DELETE leaves the resource equally deleted.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is an Idempotency-Key?	Client-generated unique ID so the server can replay a previous response.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	When to use an Idempotency-Key?	Payments, order creation, anything that must not double-execute on retry.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What does 422 Unprocessable Entity mean?	Body parsed correctly but failed validation.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	400 vs 422?	400 = cannot parse. 422 = parsed but invalid.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What does 429 Too Many Requests mean?	Rate limit exceeded. Server should send Retry-After header.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What does 502 Bad Gateway mean?	Upstream server returned an invalid response.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What does 503 Service Unavailable mean?	Server temporarily overloaded or down.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What does 504 Gateway Timeout mean?	Upstream didn't respond in time.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is Authentication?	Proving who you are.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is Authorization?	Deciding what you are allowed to do.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is a JWT?	JSON Web Token: self-contained, signed token used for stateless auth.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	JWT vs Session-Cookie?	<table border="1" cellpadding="6"><tr><th>Session</th><th>JWT</th></tr><tr><td>State on server</td><td>Stateless, self-contained</td></tr><tr><td>Easy to revoke</td><td>Hard to revoke before expiry</td></tr></table>
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is OAuth2?	Protocol for delegated authorization without sharing the password.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is an Access Token?	Short-lived token used to call protected APIs.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is a Refresh Token?	Long-lived token used to obtain new access tokens without re-login.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is CORS?	Browser security rule that controls which origins can call an API.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What triggers a CORS preflight (OPTIONS)?	Non-simple methods (PUT, DELETE), custom headers, or JSON content-type.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is offset pagination?	Skip N rows and return the next M. Simple but slow on deep pages.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	What is cursor pagination?	Continue from an opaque pointer (last seen id). Stable and fast.
AwsEngineer::SoftwareEngineering::Design::HTTPDeep	MiPlantilla	Offset vs Cursor pagination?	<table border="1" cellpadding="6"><tr><th>Offset</th><th>Cursor</th></tr><tr><td>Page jumps</td><td>Only forward, stable</td></tr><tr><td>Slow deep pages</td><td>Fast on any page</td></tr></table>

AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Monolith architecture?	Single deployable unit containing all features and one DB.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Pro of Monolith?	Simple to develop, test, and deploy.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Con of Monolith?	Hard to scale parts independently; one failure can crash all.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	When to use Monolith?	Starting out, small team, simple domain.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Microservices architecture?	System split into small independent services with their own DB.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Pro of Microservices?	Independent scaling, deploys, and tech stack per service.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Con of Microservices?	Distributed system complexity, network latency, harder to debug.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	When to use Microservices?	Large team, clear domain boundaries, need independent scaling.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Monolith vs Microservices?	<table border="1" cellpadding="6"><tr><th>Monolith</th><th>Microservices</th></tr><tr><td>One process, one DB</td><td>Many services, many DBs</td></tr><tr><td>Simple deploy</td><td>Independent deploys</td></tr></table>
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Layered architecture?	Code split into horizontal layers (Presentation → Business → Data).
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Key rule of Layered architecture?	Each layer only calls the layer directly below.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Hexagonal architecture?	Business logic isolated from external systems via ports and adapters.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is a Port in Hexagonal architecture?	Interface defined by the business logic (e.g., UserRepository).
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is an Adapter in Hexagonal architecture?	Concrete implementation of a port (e.g., PostgreSQLUserRepository).
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Benefit of Hexagonal architecture?	Business logic is testable without DB or HTTP framework.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Event-Driven architecture?	Services communicate by publishing and reacting to events.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Pro of Event-Driven architecture?	Decoupled services, resilient, scalable.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	Con of Event-Driven architecture?	Complex debugging and eventual consistency.
AwsEngineer::SoftwareEngineering::Architecture::Styles	MiPlantilla	What is Serverless architecture?	Run code on demand without managing servers (e.g., AWS Lambda).
